Pixels and Colors
An Article by John Klein

The creators of CSS implemented various methods for sizing and coloring elements in HTML. When sizing elements like fonts or margins the web developer must decide between the many kinds of absolute and relative units of length. Types of absolute length units included in CSS specifications are inches (in), centimeters (cm), millimeters (mm), points (pt), and picas (pc). One point is the same as 1/72 in and one pica is the same as 12 pt. These absolute measurements are rarely accurate in practice and thus are deprecated. This is because monitor size and resolution will alter the ratios of these so called "absolute" lengths.

Stylesheets using relative length units such as em-height (em), x-height (ex), and pixels (px) are more easily scalable from one output device to another. One em is equal to the font size declared within the element. One ex is the height of the letter x (x-height) in the font. Different fonts of the same size have different sized x's. Oftentimes browsers do not recognize the x-height of fonts so 1 ex defaults to 1/2 the font size. Traditionally, pixels are thought as absolute lengths but Eric Meyer is adamant that pixels should be defined as relative lengths:

In its discussion of pixels, the CSS specification recommends that in cases where a display type is significantly different than 96ppi [pixels per inch], user agents should scale pixel measurements to a "reference pixel." CSS2 recommended 90ppi as the reference pixel, but CSS2.1 recommends 96ppi—a measurement common to Windows machines and adopted by modern Macintosh browsers such as Safariâ€¦In general, if you declare something like font-size: 18px, a web browser will almost certainly use actual pixels on your monitor—after all, they're already there—but with other display devices, like printers, the user agent will have to rescale pixel lengths to something more sensible. In other words, the printing code has to figure out how many dots there are in a pixel, and to do so, it may use the 96ppi reference pixel. Because of this potential for rescaling, pixels are defined as a relative unit of measurement, even though, in web design, they behave much like absolute units.

Saying all this it is still up to the web designer to decide which type of length unit to size which elements of their webpage. Relative units may be more suited for elements that need to change size relative to how the text size changes whereas the more absolute pixel unit may be better suited for things like borders, absolute positioning of elements, and images.

When attempting to declare colors to elements the web designer is again faced with the decision of choosing the method that is best suited for what they are trying to accomplish. This is because CSS specifications include a variety of different ways to set color. For starters CSS2.1 defines 17 color names including aqua, fuchsia, lime, olive, red, white, black, blue, yellow to name a few. CSS3 specifies an even greater number of named colors and browsers that accommodate CSS3 will recognize up to 140 color names. However, with the use of functional RGB notation or hexadecimal notation up to 16,777,216 colors can be depicted in modern 24-bit web browsers. Let me provide some examples of how to code the same color using different notation below.

The color purple as seen in the following text can be coded the following ways:

  • as an RGB percentage: rgb(50%,0%,50%)
