Why Do Stylesheets Exist?
An Article by John Klein

The markup language of HTML was originally designed strictly for structure and little attention was given to visual presentation. However, with the advent of the Mosaic web browser the demand skyrocketed for new ways to present text, images, and website elements. Users were looking to completely customize the appearance of their websites. In order to accommodate these demands later versions of HTML haphazardly incorporated a steadily growing barrage of presentational markup. The call for stylistic freedom was met with more presentational markup. Because the structural and presentational markup were enmeshed together the "structure" of the website was no longer cut and dry. Instead, markup inside websites became a mess. Webpages were filled to the brim with presentational information in effect blurring the actual structure. The impractical outcome was that indexing content became increasingly difficult, websites were not as accessible as they could be, advanced page presentation was hindered, and maintaining websites was totally inefficient.

Eric Meyer writes:

What we need is a way to combine structural markup with attractive page presentation…Of course, the problem of polluting HTML with presentational markup was not lost on the World Wide Web Consortium (W3C), which began searching for a quick solution. In 1995, the consortium started publicizing a work-in-progress called CSS. By 1996, it had become a full Recommendation, with the same weight as HTML itself. Here's why…In the first place, CSS allows for much richer document appearances than HTML ever allowed, even at the height of its presentational fervor.

The rich styling of CSS reigns king over HTML's poor attempt. But the stylistic breadth of CSS is just the proverbial tip of the iceberg. The biggest perk to using a stylesheet is the consolidation of all presentational commands in one place. Any structural element within HTML is selectively modified by constructing a simple rule in CSS. Rather than having to style every single heading, paragraph, or text element where it arises like in the old-days of HTML, elements are simply called up once within CSS and declared a certain style. This makes editing the appearance of your site on a global scale highly efficient. HTML is now the site of structure and content while the stylesheet mandates how the content appears. Gone are the days of rummaging through cluttered HTML markup in order to add major or even minor changes to your site appearance.

As stated before the specificity of CSS "selectors" permits fine-tuning of how content/elements look. Similarly, the cascade is a powerful tool that can be used to tweak specific pages. The effect of the cascade is any stylesheet that is imported after an already in place stylesheet will override the rules of the previous stylesheet. This is a logical provision to resolve conflicts between stylesheets. Whatever stylesheet is last on the list overrules the declarations of the previous stylesheet. This feature is ideal in the situation where you want one webpage to retain most of the rules from one stylesheet that is used across your website but also want to change some already declared rules without altering the entire site. In this case create another stylesheet with the style rules you want in place of the old rules and list the stylesheet below the previous stylesheet. The web browser cascades down to the final stylesheet when rendering styles.
